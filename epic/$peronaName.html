Data-Design

User Persona:

  • Name: Beaker
  • Age: 25
  • Gender: Male
  • Ethnicity: Muppet
  • Occupation: Assistant scientist & test subject
  • Technology: Multiple technology platforms used
  • Needs, Goals, and Features :
    • Needs the ability to find parts for a wide array of technical genres
    • Electronic data sheets that are easy accessed.
    • The ability to find mating/compatible parts by there part number.

User Story

Beaker would really like to find a one stop website for all his electronic parts needs.

User Case

Title: Parts to be ordered

Description: A BOM has been generated, and now parts need to be ordered.

Users Name and Their Role: Beaker is the head scientist on a Centrifuge project.

Usage Preconditions: Should have a digikey account and log in.

Usage Prostconditions: Able to track orders when they ship and track orders in transit

Interaction Flow:

  • Beaker logs into Digi-key.
  • With the BOM he start entering part numbers to find parts.
  • When a part has been found it's add to cart.
  • Down load the datasheet for documentation.
  • Review order
  • Go to check out and place order
  • Receive email with confirmation number and shipping dates
  • Receive email with information on what shipped and tracking number.

Frequency of Use: Will be used Weekly
